Movie Recommendation System: KNN vs Matrix Factorization

Javier Kangditya

Introduction

Project Overview

The goal of this project is to build and compare two different recommendation systems for movies based on user rating behavior. The models used in this project are K-Nearest Neighbors (KNN) and Matrix Factorization (SVD). The objective is to understand how each method performs in predicting user preferences and generating personalized movie recommendations.

Dataset Description

The dataset is obtained from MovieLens and contains two main files used in this project:

  • rating.csv
    Contains user-movie interactions with the following features:
    • userId
    • movieId
    • rating
    • timestamp
  • movie.csv
    Contains movie metadata including:
    • movieId
    • title
    • genres

The dataset represents a user-item rating matrix that will be used for collaborative filtering techniques.

Project Goals

  1. Load and preprocess movie rating data
  2. Build a user-item matrix for recommendation
  3. Implement K-Nearest Neighbors (KNN) based recommender system
  4. Implement Matrix Factorization (SVD) based recommender system
  5. Evaluate and compare both models
  6. Generate movie recommendations for users
  7. Provide insights on which method performs better for sparse rating data

recall_at_k <- function(pred, actual, k = 10) {

  pred_list <- as(pred, "list")
  scores <- c()

  for (i in seq_along(pred_list)) {

    if (i > length(actual)) break

    p <- pred_list[[i]][1:k]
    a <- actual[[i]]

    if (length(a) == 0) next

    hits <- length(intersect(p, a))
    scores <- c(scores, hits / length(a))
  }

  if (length(scores) == 0) return(0)
  mean(scores)
}

map_at_k <- function(pred, actual, k = 10) {

  pred_list <- as(pred, "list")
  ap <- c()

  for (i in seq_along(pred_list)) {

    if (i > length(actual)) break

    p <- pred_list[[i]][1:k]
    a <- actual[[i]]

    if (length(a) == 0) next

    hit <- 0
    sum_prec <- 0

    for (j in seq_along(p)) {
      if (p[j] %in% a) {
        hit <- hit + 1
        sum_prec <- sum_prec + hit / j
      }
    }

    if (hit > 0) {
      ap <- c(ap, sum_prec / min(length(a), k))
    }
  }

  if (length(ap) == 0) return(0)
  mean(ap)
}

dcg <- function(rel) {
  sum(rel / log2(seq_along(rel) + 1))
}

ndcg_at_k <- function(pred, actual, k = 10) {

  pred_list <- as(pred, "list")
  scores <- c()

  for (i in seq_along(pred_list)) {

    if (i > length(actual)) break

    p <- pred_list[[i]][1:k]
    a <- actual[[i]]

    if (length(a) == 0) next

    rel <- as.numeric(p %in% a)
    ideal <- sort(rel, decreasing = TRUE)

    dcg_val <- dcg(rel)
    idcg_val <- dcg(ideal)

    if (idcg_val == 0) next

    scores <- c(scores, dcg_val / idcg_val)
  }

  if (length(scores) == 0) return(0)
  mean(scores)
}

Conclusion & Insights

1. Summary of Findings

  • Recommendation System Evaluation:
    • Three models were evaluated: POPULAR, Matrix Factorization (MF), and K-Nearest Neighbors (KNN).
    • Evaluation metrics used were Recall@10, MAP@10, and NDCG@10 to measure both retrieval quality and ranking performance.
    • The dataset shows a strong popularity bias, where a small number of frequently rated items dominate user preferences.
  • Model Performance:
    • The POPULAR model achieved the best overall performance across all metrics, especially in MAP@10 and NDCG@10.
    • The MF model showed moderate performance but was limited by data sparsity and insufficient interaction density.
    • The KNN (IBCF) model performed the weakest, indicating that user-item similarity is not stable in sparse conditions.
  • Evaluation Insights:
    • Recall@10 values are relatively low, which is expected in recommender systems with a large item space.
    • MAP@10 and NDCG@10 provide better insight into ranking quality, showing that POPULAR consistently ranks relevant items higher.
    • Overall, simpler baseline models can outperform more complex models depending on dataset characteristics.

2. Practical Implications

  • Popularity-based recommendation is highly effective in datasets with strong item popularity bias.
  • Matrix Factorization is more suitable when user-item interactions are denser and more balanced.
  • KNN-based collaborative filtering is sensitive to sparsity and may not perform well without sufficient data.
  • In real-world systems, baseline models like POPULAR remain strong benchmarks and should not be ignored.

3. Recommendations

  • For production systems with sparse data, start with a POPULAR baseline as a strong benchmark.
  • Improve MF performance by tuning hyperparameters (latent factors, regularization, iterations).
  • Consider hybrid recommendation approaches (e.g., combining POPULAR + MF) to improve robustness.
  • Collect more interaction data to improve collaborative filtering performance, especially for KNN-based models.
